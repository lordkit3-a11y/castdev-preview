# Правила: кастдевы, транскрибации, сайт

Документ для автора и для ассистента. Перед добавлением нового кастдева **открой этот файл** и пройди чеклист.

## 1. Нумерация и имена файлов

Один созвон = один номер `NN` (двузначный: `01`, `02`, …).

| Что | Шаблон имени | Пример |
|-----|----------------|--------|
| Транскрибация (сырая расшифровка) | `NN_Транскрибация_Имя_Клиент.txt` | `04_Транскрибация_Андрей_Каторгин_Skoro.market.txt` |
| HTML кастдева | `NN_Кастдев_Имя_Клиент.html` (тот же `NN`) | `04_Кастдев_Андрей_Каторгин_Skoro.market.html` |

Правила:

- В имени файла пишем **`Транскрибация`** (как в папке), не «Транскрипция», чтобы все файлы искались одним шаблоном.
- Если пользователь положил файл **без номера** или с «транскрипцией» в названии — **агент переименовывает** в соответствии с таблицей и следующим свободным `NN`.
- `NN` одинаковый у пары «транскрибация + html».

## 2. Куда класть файлы

- Транскрибации → `Работа с кастдевами/Транскрибации/`
- Публичный сайт (что уходит на GitHub Pages) → `Работа с кастдевами/Сайт/`
- Служебное (css, js, json) → `Работа с кастдевами/Сайт/Служебные/`

## 3. После нового HTML кастдева

1. Добавить строку в таблицу на главной: `Работа с кастдевами/Сайт/index.html`
2. Добавить блок задач в секцию «Задачи по кастдевам» (те же `data-task-id`, что на странице кастдева)
3. При необходимости — кнопка «Запись созвона» (как у других кастдевов)

## 4. Что НЕ считать «дублями» (корень vs `Сайт/`)

В **корне** лежат **короткие** `index.html` и `kastdev-*.html` — это **редиректы** на полные файлы в `Работа с кастдевами/Сайт/`. Размеры и HTML **не совпадают** с версиями в `Сайт/` (проверка: `wc -c` или `cmp`). Это **не дубли контента**.

Переносить корневые редиректы в `дубли` **нельзя** — сломаются короткие URL на GitHub Pages.

Отдельно: в корне один **`README.md`** для всего репо. В `Сайт/Служебные/README.md` — только ссылка на корень и на этот файл правил.

## 5. Папка `дубли`

`Работа с кастдевами/дубли/` — для **устаревших или полных копий** файлов, которые сознательно убраны из рабочих путей (например, дубль после переименования). Не класть туда рабочие транскрибации и актуальные HTML.

Отдельно: папка `castdev-public` в каталоге `Cursor` (рядом с `CustDev`), если она ещё есть — это **старый отдельный репозиторий**; с основным репо `CustDev` не смешивать и не дублировать без согласования.
