# Каздев — превью для GitHub Pages

**Правила (нумерация, имена, дубли, что не трогать):** `Работа с кастдевами/ПРАВИЛА_кастдевы.md`

## Структура папки `CustDev`

- **`Примеры`** — ориентиры и шаблоны. Не участвуют в текущей работе, только как образец.
- **`Работа с кастдевами/Транскрибации`** — исходные расшифровки созвонов.
- **`Работа с кастдевами/Сайт`** — финальные HTML: `index.html` и страницы кастдевов.
- **`Работа с кастдевами/Сайт/Служебные`** — техническое для сайта: `json`, `js`, `css`, инструкции.
- **`Работа с кастдевами/дубли`** — архив/лишние копии (см. `README` внутри); рабочие файлы тут не храним.

### Именование

- Транскрибация: `NN_Транскрибация_Имя_Клиент.txt`
- HTML кастдева — с тем же номером, что и транскрибация.
- Новый кастдев: сначала транскрибация в `Транскрибации` с **следующим** `NN`, затем HTML в `Сайт` с тем же `NN`.

### GitHub Pages

**Боевой сайт** — только в `Работа с кастдевами/Сайт/`. В **корне** репозитория лежит `index.html` с **редиректом** на эту папку (чтобы `https://…github.io/…/preview/` открывал актуальную главную). Короткие ссылки `…/kastdev-roman-…html` и `…/kastdev-ssk-…html` в корне — тоже **редиректы** на файлы внутри `Сайт/`.

У **GitHub Pages** источник — **корень ветки** `main` (или `/docs`). Отдельно настраивать подпапку не нужно: редирект ведёт в `Сайт/`.

### Обновление (из корня репозитория)

```bash
cd "/Users/igor/Cursor/CustDev"
git add "Работа с кастдевами/Сайт/index.html" "Работа с кастдевами/Сайт/kastdev-roman-dars-2026-04-08.html"
git commit -m "Обновление кастдева"
git push
```

Новый кастдев: новый `.html` в `Работа с кастдевами/Сайт/` + строка в таблице в `index.html`.
